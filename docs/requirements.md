# 要件定義書

## プロジェクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | MyTrelloアプリ１号 |
| 目的 | 個人のタスクを視覚的に管理する |
| 利用者 | 1人（個人利用） |
| 作成日 | 2026年6月10日 |

---

## 背景・目的

本アプリはプログラミングスクールの課題として作成するものであり、主な目的は以下の学習にある。

- React を用いたフロントエンド開発の実践
- Java Spring Boot を用いたバックエンドAPI開発の実践
- PostgreSQL を用いたデータベース設計・操作の理解
- Gradle によるJavaプロジェクトのビルド管理
- フロントエンドとバックエンドを連携したフルスタック開発の体験

成果物としてはTrelloライクなタスク管理ボードを作成し、上記技術を組み合わせて動作するWebアプリを完成させる。

## 非機能要件

| 項目 | 内容 |
|------|------|
| データ保存 | Spring Boot経由でPostgreSQLに保存 |
| 対応環境 | PCのWebブラウザ（Chrome推奨） |
| レスポンシブ | 対象外（PC画面のみ） |
| ログイン機能 | 不要（個人利用のため） |

---

## 制約・前提条件

| 種別 | 内容 |
|------|------|
| 動作環境 | PCのChrome最新版での動作を前提とする。他ブラウザでの動作は保証しない |
| サーバー起動 | アプリ利用時はSpring Boot（Java）サーバーとPostgreSQLをローカルで起動している必要がある |
| ネットワーク | フロントエンドとバックエンドが同一ローカル環境で動作することを前提とする |
| データ共有 | 同一端末内のみ。他端末とのデータ同期は対象外 |
| 複数タブ | 複数タブで同時に開いた場合のデータ整合性は保証しない |

---

## 対象外（今回は作らないもの）

- 複数人でのデータ共有
- スマートフォン対応
- ログイン・ユーザー管理
- 列の追加・削除・名前変更
- タスクの検索・フィルター機能
- 本番環境へのデプロイ
