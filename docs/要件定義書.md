# 要件定義書

## プロジェクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | MyTrelloアプリ１号 |
| 目的 | 個人のタスクを視覚的に管理する |
| 利用者 | 1人（個人利用） |
| 作成日 | 2026年6月10日 |

---

## 機能要件

機能要件とは「このアプリが何をできるか」を定義するものです。

### 1. ボード（画面全体）

- 画面を開くと、3つの列が横に並んで表示される
- 列の構成は固定（変更・追加・削除はできない）

### 2. 列（リスト）

| 列名 | 役割 |
|------|------|
| 未着手 | まだ手をつけていないタスク |
| 進行中 | 現在取り組んでいるタスク |
| 完了 | 終わったタスク |

### 3. カード（タスク）

カード1枚が1つのタスクを表す。

**カードに含まれる情報**

| 項目 | 必須/任意 | 内容 |
|------|-----------|------|
| タイトル | 必須 | タスクの名前 |
| 説明文 | 任意 | タスクの詳細メモ |
| 期限日 | 任意 | いつまでに終わらせるか |
| 優先度 | 任意 | 高・中・低の3段階 |

**カードに対してできる操作**

| 操作 | 説明 |
|------|------|
| 追加 | 各列の「＋」ボタンからカードを新規作成する |
| 編集 | カードをクリックして内容を変更する |
| 削除 | カード上の「削除」ボタンで削除する |
| 移動 | ドラッグ&ドロップで別の列に移動する |

---

## 非機能要件

非機能要件とは「どのように動くか」を定義するものです。

| 項目 | 内容 |
|------|------|
| データ保存 | ブラウザのlocalStorageに保存（サーバー不要） |
| 対応環境 | PCのWebブラウザ（Chrome推奨） |
| レスポンシブ | 対象外（PC画面のみ） |
| ログイン機能 | 不要（個人利用のため） |

## 対象外（今回は作らないもの）

- 複数人でのデータ共有
- スマートフォン対応
- ログイン・ユーザー管理
- 列の追加・削除・名前変更
- タスクの検索・フィルター機能

---

## 使用技術

| 役割 | 技術 |
|------|------|
| 画面の構造 | HTML |
| 見た目・デザイン | CSS |
| 動き・データ保存 | JavaScript |
| ドラッグ&ドロップ | HTML5標準のDrag and Drop API |
| データ保存先 | localStorage（ブラウザ内） |
